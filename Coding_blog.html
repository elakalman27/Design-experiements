Week 2

Topic: HTML basics (1)

This lecture covered HTML basics. This was easy to understand and get the hang of. Writing the code and seeing the results instantly on the open tab was very exciting. This feels like getting to know a new secret world that is hidden from most internet users. My main concern is remembering all the tags but that will come with time. I first struggled to place the image through the link filing way, it took me a few tries but eventually, I realised I was putting the end closing bracket within the “”. This is very exciting and makes me want to learn more about code, it feels achievable. Although I am aware that the knowledge between creating the basic lines of text to a fully realised, functional design is probably vast.

Week 3

Topic: HTML basics (2)

New terms:
- Anchor tag- wraps around a word or an image to make it into a link
- List tags- Unordered list (ul), (li), ordered list (ol)
- Spacing tags- break (br), height and width are just height and width

Part 2 of HTML basics shows how to use anchor tags and create lists. Combining last week’s image placing and title skills with this week’s lists gives me the basic knowledge to create a site. The first time I went through the lecture I struggled a bit, both to understand the tags and to follow the order of things. In code, there is absolutely no space for error which can be very confusing when you have no idea what you’re doing. In my first attempt, I created a site ranking cats and it went mostly well but one of the images wouldn’t show up and I couldn’t figure out how to wrap the anchor tag around the image. The second time, I made a site ranking Dusty Springfield songs and although I had minor issues figuring out where to place closing tags, it all worked out. I’ve gained a good understanding of the basics now and I think I will be able to take it to the next level, attending to the visuals of the sites.

Week 5 (2?)

Topic: CSS Grids part 1

Notes:
- The header, main and footer are HTML’s default ways of adding sections
- VH and VW are dependent on screen size
- We created grid within body {}, then defined grid template area as header, main and footer, then link the header main and footer from the index by writing header {grid-area: header) etc.
- We can create grids within each section: display: grid; within the section
- To create areas on row, type areas on the same line within the same “”
-To let css know which tag is which when there are identical tags, type: nav: nth-of-type(1)

Learning how grids work is code is so interesting as it’s bridging the gap between the mechanics and visuals of site-building. It makes so much sense and in a way, it’s even easier than CMS in the way that you have total control over the visuals as long as you know where to type what. Writing the code alongside the website is very satisfying. I will still need to familiarize myself further with Flexbox terms and tags so the workflow can be smoother.
